## TL;DR

The standard project workflow chains skills in order: using-superpowers → brainstorming → planning-with-files → grill-me → writing-plans → executing-plans.

## Details

This is the high-level workflow for taking a project from idea to implementation using the superpowers skill chain:

| Step | Skill | Purpose |
|------|-------|---------|
| 1 | `using-superpowers` | Entry point — routes to the right skill based on what you're doing |
| 2 | `/brainstorming` | Explore the idea, ask clarifying questions, propose approaches, produce a design spec |
| 3 | `/planning-with-files` | Turn the design into a file-level plan (what goes where) |
| 4 | `/grill-me` | Stress-test the plan — challenge assumptions, find gaps |
| 5 | `/writing-plans` | Produce a detailed, step-by-step implementation plan |
| 6 | `/executing-plans` | Execute the plan task-by-task with verification |

This workflow can be expanded later with additional steps or branching paths (e.g., code review, testing, deployment).

## When This Matters

When starting any non-trivial project with Claude Code. Following this sequence ensures you don't jump to implementation before the design is solid, and that the plan gets stress-tested before execution.
