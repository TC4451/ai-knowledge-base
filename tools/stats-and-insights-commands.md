## TL;DR

Two built-in Claude Code slash commands for understanding your own usage: `/stats` shows a visual calendar graph and usage data points; `/insights` generates a full HTML report that analyzes your usage patterns and recommends changes to CLAUDE.md, skills, hooks, and more.

## Details

**`/stats`** — shows your own usage data, including a visual calendar graph and other
usage data points. Quick way to see your activity at a glance.

**`/insights`** — generates a full HTML page that:
- analyzes and categorizes how you use Claude Code
- describes your usage patterns
- recommends changes based on those patterns — CLAUDE.md tweaks, skills to add,
  hooks to set up, context changes, and other features you may not be using yet

Use `/insights` when you want to know whether you're using Claude Code to its full
potential or could benefit from more features.

## When This Matters

Run `/stats` when you're curious about your activity. Run `/insights` when you want a
concrete, personalized audit of your setup with recommended improvements — a good
periodic check to catch features, skills, or config you're not taking advantage of.
